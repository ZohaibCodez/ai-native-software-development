# Lesson 1: Understanding Decisions

## Opening Hook

Your favorite restaurant app needs to decide: Is it still open? Is it within your delivery radius? Should it show a discount? Every "yes" or "no" question a program answers is a decision. ## Decisions Everywhere

Every app you use makes decisions, thousands per second. Your email app decides: Is this message spam? Your game decides: Did the player reach the goal? Your bank decides: Can this account withdrawal go through?

Here's the pattern: Every decision has a **condition** (a question) and an **action** (what happens if true).

## True and False: The Foundation of Decisions

Every condition in programming evaluates to exactly one of two values: **True** or **False**. Nothing in between. When you ask a program "Is 5 greater than 3?" it answers True. When you ask "Is my name Bob?" (and it's Alice), it answers False.

These True/False values are called **Booleans** in programming (named after mathematician George Boole).

**Simple examples:**

```python
# Testing conditions - each evaluates to True or False
age: int = 17
can_drive: bool = age >= 16  # True (17 is greater than or equal to 16)

score: int = 72
is_passing: bool = score >= 70  # True (72 is greater than or equal to 70)

name: str = "Alice"
is_bob: bool = name == "Bob"  # False (name is not "Bob")

temperature: int = 82
is_hot: bool = temperature > 80  # True (82 is greater than 80)
```

Notice the pattern: Each comparison produces a **bool** (True or False).

## Comparison Operators: Tests That Return True or False

Python gives you six comparison operatorsâ€”each tests a different relationship between two values:

| Operator | Meaning | Example |
|----------|---------|---------|
| `>` | greater than | `5 > 3` evaluates to True |
| `<` | less than | `5 < 3` evaluates to False |
| `==` | equal to | `5 == 5` evaluates to True |
| `!=` | not equal to | `5 != 3` evaluates to True |
| `>=` | greater than or equal to | `5 >= 5` evaluates to True |
| `<=` | less than or equal to | `5 <= 3` evaluates to False |

Each operator **tests** a relationship and returns True or False.

**Worked examples:**

```python
# Test each operator
num1: int = 10
num2: int = 5

print(num1 > num2)   # True (10 is greater than 5)
print(num1 < num2)   # False (10 is not less than 5)
print(num1 == num2)  # False (10 is not equal to 5)
print(num1 != num2)  # True (10 is not equal to 5)
print(num1 >= num2)  # True (10 is greater than or equal to 5)
print(num1 <= num2)  # False (10 is not less than or equal to 5)
```

**With strings:**

```python
name1: str = "Alice"
name2: str = "Bob"

print(name1 == name2)  # False (different names)
print(name1 != name2)  # True (they are different)
```
